# 계산 로직 설명

## 1. 학교 티어 결정

- 사용자가 학교를 선택하면 `고등학교DB`에서 해당 학교의 인문/자연 티어를 찾습니다.
- 지원 행의 `계열` 값이 `자연`이면 자연 티어를, 그 외에는 인문 티어를 사용합니다.
- 이 값이 엑셀 `계산기!M` 역할을 대체합니다.

## 2. 학생 점수 결정

- `계산기!N` 열의 수식을 읽어 어떤 입력 셀을 참조하는지 확인합니다.
- 참조식 문자열을 그대로 실행하지 않고 `성적입력!$C$15 -> grades.coreAll` 같은 매핑 테이블로 변환합니다.
- 현재 버전에서 확인된 기본 데이터는 실질적으로 `성적입력!C15`를 사용합니다.

## 3. 배치표 재구성

### 공통 규칙

- 보정값 시트의 각 블록은 다음 요소를 가집니다.
- 티어별 가감점
- 티어별 상향/안정 offset
- `인문1/2/3`, `자연1/2/3`의 기준 fit 값
- 티어별 gap 규칙

### 재구성 공식

- 기본형: `fit = baseFitScore + tierBonus`
- `reach = fit + reachOffset`
- `safe = fit + safeOffset`

### 인문1/3, 자연1/3 특수 규칙

- `인문1/3`, `자연1/3`은 같은 계열의 `2` 기준값을 먼저 찾습니다.
- 티어가 `C`이고 직접 기준값이 존재하면 직접 기준값을 우선 사용합니다.
- 그렇지 않으면 `2` 기준값에 티어 가감점과 gap을 반영합니다.
- `1`은 gap을 빼고, `3`은 gap을 더합니다.
- `2` 기준값이 없으면 직접 기준값 + 티어 가감으로 fallback 합니다.

## 4. 판정 로직

- 컷이 하나라도 비어 있으면 판정은 빈 문자열로 처리합니다.
- `score <= safe` 이면 `안정`
- `score <= fit` 이면 `적정`
- `score <= reach` 이면 `소신`
- 그 외는 `상향`

## 5. 오류 처리 방식

- 엑셀의 `IFERROR(..., "")`는 TypeScript에서 `null` 또는 빈 문자열 fallback으로 바꿨습니다.
- 찾기 실패, 수치 없음, 기준표 누락은 모두 빈 threshold로 처리합니다.
- 비교 시에는 부동소수 오차를 줄이기 위해 작은 epsilon을 사용합니다.

## 6. 검증 결과 해석

- 배치표 재구성은 종합 148건, 교과 526건 모두 0 mismatch였습니다.
- 결과행 비교에서는 26건 mismatch가 남았지만, 모두 stale/inconsistent Excel cached row로 분류했습니다.
- 즉 현재 확인 범위에서는 재구성 로직 mismatch가 아니라 원본 엑셀의 캐시 잔존 문제입니다.
